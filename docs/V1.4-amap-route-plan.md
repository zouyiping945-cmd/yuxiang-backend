# V1.4 高德路线规划接入说明

## 1. V1.4 目标

V1.4 将 Result 页“去这里”从静态建议升级为真实路线摘要。用户点击后，小程序尝试获取当前位置，请求后端 `/api/route`，由后端读取村庄坐标并调用高德路径规划，最终返回距离、耗时、路线建议和出行提醒。

## 2. 为什么先做路线摘要

本阶段先使用高德 Web Service 路径规划，而不直接接入地图 SDK。这样可以继续沿用现有弹窗交互，不引入地图组件、前端 Key 和额外权限配置，同时把高德 Key 保留在服务端，降低前端泄露风险。

## 3. `/api/route` 请求结构

接口：

```text
POST /api/route
```

请求示例：

```json
{
  "villageId": "zhengzhou_yingtaogou",
  "origin": {
    "longitude": 113.625368,
    "latitude": 34.746599
  },
  "mode": "driving"
}
```

`destination` 可选。前端有目标坐标时可以直接传入；没有时，后端根据 `villageId` 查询 Supabase `villages` 表的 `longitude / latitude`。

## 4. `/api/route` 返回结构

成功时返回：

```json
{
  "ok": true,
  "data": {
    "provider": "amap",
    "mode": "driving",
    "originUsed": {
      "longitude": 113.625368,
      "latitude": 34.746599,
      "source": "user"
    },
    "destinationUsed": {
      "longitude": 113.589043,
      "latitude": 34.617794,
      "name": "樱桃沟社区"
    },
    "distanceMeters": 28000,
    "distanceText": "约28公里",
    "durationSeconds": 2700,
    "durationText": "约45分钟",
    "summary": "建议自驾前往，全程约28公里，预计约45分钟。周末建议错峰出发，抵达前确认停车位置。",
    "tips": [
      "建议提前确认停车点",
      "老人同行建议减少夜间山路驾驶"
    ],
    "fallbackUsed": false
  }
}
```

高德、环境变量、坐标或请求解析失败时，接口仍返回 HTTP 200 与 `ok: true`，但 `provider` 为 `fallback`、`fallbackUsed` 为 `true`，便于前端继续展示静态建议。

## 5. 高德路径规划调用方式

后端通过以下接口规划 V1.4 的自驾路线：

```text
GET https://restapi.amap.com/v3/direction/driving
```

关键参数：

```text
origin=longitude,latitude
destination=longitude,latitude
key=AMAP_WEB_SERVICE_KEY
extensions=base
```

请求超时为 10 秒。后端只解析首条路线的 `distance` 和 `duration`，不把高德完整路线对象返回前端。

## 6. 用户定位授权策略

Result 页使用 `Taro.getLocation({ type: "gcj02" })` 尝试获取当前位置。定位不是强制权限：授权成功时将当前位置作为 `origin`；拒绝授权或获取失败时继续请求后端，不阻塞“去这里”流程。

## 7. 默认起点 fallback 策略

前端没有传入有效 `origin` 时，后端使用以下默认起点估算：

```json
{
  "longitude": 113.625368,
  "latitude": 34.746599,
  "source": "default"
}
```

返回结果中的 `originUsed.source = "default"`，前端据此提示“已使用默认起点估算路线”。

## 8. 高德失败 fallback 策略

以下情况统一回落到 fallback 结果，不返回 500：

- 缺少 `AMAP_WEB_SERVICE_KEY`
- 缺少 Supabase 服务端配置
- 村庄坐标不可用
- 高德请求超时或 HTTP 失败
- 高德返回 `status != 1`
- 高德未返回有效路径
- 请求 JSON 无法解析

服务端日志分别记录 `amap route success` 或 `amap route fallback reason`。

## 9. 前端“去这里”展示

点击“去这里”后显示“正在规划路线...”加载态。请求完成后沿用原弹窗，展示：

- 出行方式
- 距离
- 耗时
- 路线建议
- 出行提醒

若使用默认起点，会显示默认起点提示；若 `fallbackUsed = true`，会显示“路线为估算建议，请出发前再次确认”。网络请求失败时继续使用原静态路线建议。

## 10. 下一阶段 V1.5

V1.5 计划读取 `village_pois` 中已审核且推荐的真实美食、住宿 POI，在 Result 页展示真实店铺，并让 DeepSeek 基于真实 POI 生成规划解释。
