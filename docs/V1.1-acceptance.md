# V1.1 DeepSeek 智能体雏形验收记录

## 1. 当前状态

- V1.1 已完成。
- 后端已接入 DeepSeek API。
- Netlify 线上后端已配置 DeepSeek 环境变量。
- 线上 `/api/plan` 已测试通过。
- 返回 `providerUsed = deepseek`。
- 返回 `fallbackUsed = false`。
- `reasonTags / reasonSummary / travelTips` 正常返回。
- `recommended / alternatives` 仍来自真实村庄数据和规则推荐结果。

## 2. 架构变化

当前链路：

```text
用户输入需求
↓
读取 Supabase / REAL_VILLAGES
↓
规则打分确定 recommended 和 alternatives
↓
DeepSeek 基于候选村庄生成推荐解释
↓
返回 summary / reasonTags / reasonSummary / travelTips
```

说明：

DeepSeek 不直接决定村庄 id，不允许编造不存在的村庄。

## 4. 环境变量

本地和 Netlify 均需配置：

```text
AI_PROVIDER=deepseek
DEEPSEEK_API_KEY
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat
```

仍需保留：

```text
SUPABASE_URL
SUPABASE_SERVICE_ROLE_KEY
```

说明：

`DEEPSEEK_API_KEY` 和 `SUPABASE_SERVICE_ROLE_KEY` 只能放后端环境变量，不放小程序端。

## 5. fallback 策略

- DeepSeek 失败时 fallback 到规则推荐。
- Supabase 失败时 fallback 到 `REAL_VILLAGES`。
- `/api/plan` 不应因为 AI 或数据库失败直接 500。
- 小程序端不需要知道 fallback 细节。

## 6. 当前限制

- Result 页目前不一定完整展示 `reasonTags / reasonSummary / travelTips`。
- 首页还不是完全 AI 主入口。
- 还没有多轮对话。
- 还没有真正的工具调用式 Agent。
- 当前仍是智能体雏形。

## 7. 下一阶段 V1.2 建议

V1.2 建议：

- 重构首页和 AI 页，让 AI 成为主入口。
- 增加更明显的 AI 规划助手表达。
- 展示 DeepSeek 返回的 `reasonTags / reasonSummary / travelTips`。
- 支持用户继续调整需求，例如更近一点、不爬山、更适合父母、想拍照、住一晚。
