# V0.6 郑州圈真实 Seed 村庄扩展记录

## 1. V0.6 目标

V0.6 的目标是在不改变前端 UI、不接 Supabase、不修改 `/api/plan` 返回结构的前提下，将真实村庄 seed 数据从 V0.5 的 8 个郑州及近郊样本扩展到 20-30 个郑州圈样本。

本轮重点不是做完整数据库，而是让 `/api/plan` 有更丰富的郑州圈真实候选池，支持 Result 页的推荐、换一个、查路线、觅美食、寻住处和我的行程保存继续稳定运行。

## 2. 为什么先扩展郑州圈

当前产品主链路仍处于 MVP 阶段，用户测试和演示场景集中在郑州本地与近郊周末出游。

先扩展郑州圈有几个好处：

- 数据范围更可控，便于人工核验。
- 推荐结果更贴近当前用户定位和使用场景。
- 能覆盖父母同行、亲子、朋友拍照、自驾短途、传统村落、农家菜等核心需求。
- 后续迁移 Supabase 时，可以先用郑州圈数据验证表结构和推荐字段。

## 6. 去重规则

V0.6 使用以下规则避免重复建库：

- `id` 必须唯一。
- 已存在的 8 个 seed 不重复添加。
- 神仙洞村不同名单写法合并到 `zhengzhou_shenxiandong`。
- 乡镇或街区类条目不强行当作普通自然村；例如“大金店老街”保留为单条 seed，但 `placeLevel` 标记为 `unknown`，后续人工核验。
- 同一个村庄如果后续发现多来源，优先在 `designations` 中追加来源，不新建重复村庄。

## 7. 数据仍处于 seed 阶段

当前 24 个样本仍是 seed 数据：

- 餐饮名称为保守通用文案，不代表真实商家。
- 住宿名称为保守通用文案，不代表真实民宿。
- 价格为估算表达，如“人均约 ¥45”“¥198起”“需提前确认”。
- 路线时长为产品级占位建议，后续需要地图或人工核验。
- 不返回远程图片，不使用 Unsplash。
- 不包含坐标、电话、营业时间等需核验字段。

## 8. 推荐逻辑更新

V0.6 保持 V0.5 的关键词打分逻辑，并补充了自驾意图识别。

当前会识别：

- 老人 / 父母 / 不累 / 轻松
- 亲子 / 孩子
- 农家菜 / 美食
- 拍照 / 出片 / 风景
- 康养 / 放松
- 研学 / 文化 / 传统村落
- 周末 / 短途 / 近郊
- 自驾 / 开车 / 驾车

新增 seed 进入 `REAL_VILLAGES` 后会自动参与排序，`alternatives` 也从完整真实候选池中选择，不再限制在原有 8 个样本中。

## 9. 下一阶段 V0.7 建议
