# V1.3 高德地理坐标与 POI 基础能力接入方案

## 1. V1.3 目标

V1.3 只建设数据基础能力，不改前端 UI，不改 `/api/plan` 返回结构，也不改变 DeepSeek 与 Supabase 的推荐链路。

本轮目标是为现有 24 个郑州圈 seed 村庄补充：

- 高德坐标候选
- adcode 行政区划编码
- amap_poi_id 预留字段
- 周边 food / stay / parking POI 候选

这些数据全部先标记为 `needs_review`，不直接作为已验证商家或已验证坐标使用。

## 2. 为什么先补经纬度

路线规划、距离计算、周边 POI、地图展示都依赖稳定坐标。

当前村庄 seed 数据已有名称、行政区、推荐标签和出行画像，但缺少统一坐标。先补经纬度可以为后续这些能力打基础：

- V1.4 真实路线摘要
- V1.5 Result 页读取真实 POI
- 后续地图导航、周边距离、路线耗时
- 人工核验数据后台

## 3. 高德 geocode 的作用

`scripts/amap-geocode-villages.ts` 会读取 Supabase `villages` 表，根据：

- province
- city
- district
- town
- village / name / full_name

拼出结构化地址，调用高德地理编码接口生成坐标候选，并写回：

- address
- latitude
- longitude
- adcode
- geo_source = `amap`
- geo_review_status = `needs_review`
- geo_review_notes

如果某条村庄已经是 `geo_review_status = verified` 且已有坐标，脚本会跳过，避免覆盖人工核验结果。

## 4. 高德 POI 搜索的作用

`scripts/amap-sync-village-pois.ts` 会基于已拥有坐标的村庄，搜索周边候选 POI。

本轮只抓三类：

- food：餐饮 / 农家菜 / 饭店
- stay：住宿 / 民宿 / 酒店 / 客栈
- parking：停车场

写入 `village_pois` 后，所有候选均为：

- source = `amap`
- data_review_status = `needs_review`
- is_recommended = `false`

脚本不会把高德候选直接设置为推荐店铺。

## 5. 为什么高德结果先标记为 needs_review

高德搜索结果可能存在：

- POI 名称相似但不是目标村庄周边
- 距离合理但实际不可达
- 店铺已关闭或信息过期
- 酒店 / 民宿 / 农家菜分类不完全准确
- 坐标落点在行政区中心而非村庄实际位置

所以本轮只作为“候选数据层”，后续需要人工核验后才可进入前端正式展示。

## 6. 新增字段说明

`villages` 表新增字段：

- `address`：高德返回或人工整理的地址
- `latitude`：高德坐标系纬度
- `longitude`：高德坐标系经度
- `adcode`：行政区划编码
- `amap_poi_id`：高德 POI ID 预留字段
- `geo_source`：坐标来源，例如 `amap` / `manual` / `seed`
- `geo_review_status`：坐标核验状态，`needs_review` / `verified` / `rejected`
- `geo_review_notes`：坐标核验备注

## 7. village_pois 表说明

`village_pois` 用于存储每个村庄周边 POI 候选。

核心字段：

- `village_id`：关联 villages
- `poi_id`：高德 POI ID
- `category`：food / stay / parking / activity
- `name`：POI 名称
- `address`：POI 地址
- `latitude` / `longitude`：POI 坐标
- `distance_meters` / `distance_text`：距离
- `type_text`：高德分类文本
- `tel`：电话
- `rating`：评分
- `price_text`：价格文本
- `raw`：高德原始数据
- `is_recommended`：是否推荐展示
- `data_review_status`：数据核验状态
- `review_notes`：人工核验备注

唯一约束：

`unique(village_id, poi_id, category)`

## 8. 执行顺序

1. 在 Supabase SQL Editor 执行：

```text
docs/supabase-v1.3-geo-poi.sql
```

2. 在后端 `.env.local` 配置：

```text
AMAP_WEB_SERVICE_KEY=
```

3. 运行坐标候选脚本：

```powershell
cd E:\乡豫app\原型图
npx tsx scripts/amap-geocode-villages.ts
```

4. 人工检查坐标候选。

5. 运行 POI 候选同步脚本：

```powershell
npx tsx scripts/amap-sync-village-pois.ts
```

6. 运行检查脚本：

```powershell
npx tsx scripts/check-village-geo-poi.ts
```
