# V0.8 Supabase 表结构设计

## 1. V0.8 目标

V0.8 只做 Supabase 表结构设计和迁移准备，不做真实数据迁移。

本轮不接 Supabase、不改 `/api/plan` 读取逻辑、不改前端 UI、不运行 npm。当前后端仍继续优先使用本地 `REAL_VILLAGES`。

## 2. 为什么现在开始设计 Supabase

现在开始设计 Supabase 的原因：

- 24 个 seed 已经具备迁移价值。
- `RealVillageData` 字段已经相对稳定。
- 后续扩展到 100+ 村庄需要数据库支撑。
- 推荐算法已经开始依赖结构化字段，如适老、亲子、拍照、农家菜、文化、康养、自驾、强度和建议时长。
- 提前设计表结构，可以避免后续迁移时把所有字段堆进单表。

## 3. 表结构总览

V0.8 设计 6 张表：

1. `villages`
2. `village_profiles`
3. `village_designations`
4. `village_routes`
5. `village_foods`
6. `village_stays`

设计思路：

- `villages` 保存主村庄信息。
- `village_profiles` 保存推荐画像和结构化匹配字段。
- `village_designations` 保存官方名单来源。
- `village_routes` 保存路线建议。
- `village_foods` 保存美食建议。
- `village_stays` 保存住宿建议。

## 4. 每张表字段说明

### villages

主村庄表。

字段用途：

- `id`：数据库主键。
- `village_code`：本地 seed 的稳定业务 id，对应 `RealVillageData.id`。
- `name`：村庄或目的地名称。
- `province / city / district / town / village`：行政区划。
- `full_name`：完整行政名称。
- `place_level`：地点层级，如 village、community、town、scenic_area、unknown。
- `rating`：当前展示评分，seed 阶段为产品展示字段。
- `distance_text`：距离描述。
- `description`：村庄介绍。
- `visit_duration`：建议游玩时长。
- `intensity`：游玩强度，低 / 中 / 高。
- `source_confidence`：来源可信度。
- `data_status`：数据阶段，如 seed、enriched、verified。
- `data_review_status`：人工核验状态。
- `review_notes`：人工核验备注。
- `created_at / updated_at`：时间戳。

### village_profiles

推荐画像表。

字段用途：

- `village_id`：关联 `villages.id`。
- `tags`：前端展示和推荐使用的标签。
- `suitable_for`：适合人群。
- `match_keywords`：自然语言匹配关键词。
- `recommended_transport`：推荐交通方式。
- `elder_friendly`：是否适合老人。
- `kid_friendly`：是否适合亲子。
- `photo_friendly`：是否适合拍照。
- `food_friendly`：是否适合农家菜 / 美食。
- `culture_friendly`：是否偏传统文化 / 研学。
- `wellness_friendly`：是否偏康养放松。
- `self_drive_friendly`：是否适合自驾。
- `public_transport_friendly`：公共交通是否相对可行。

### village_designations

官方名单来源表。

字段用途：

- `village_id`：关联 `villages.id`。
- `designation_type`：名单类型，如乡村旅游重点村、美丽休闲乡村、传统村落、康养旅游村。
- `source_name`：来源名称，如《附录1 河南省乡村旅游地名单》。
- `source_index`：来源索引或原表序号，当前可为空。
- `note`：备注。

### village_routes

路线表。

字段用途：

- `village_id`：关联 `villages.id`。
- `title`：路线标题，如“自驾约45分”。
- `subtitle`：路线说明。
- `icon`：展示图标文案，如“车”“巴”。
- `sort_order`：排序。

### village_foods

美食表。

字段用途：

- `village_id`：关联 `villages.id`。
- `name`：美食推荐名称。
- `desc`：描述。
- `price_text`：价格文案。
- `tag`：标签。
- `sort_order`：排序。

### village_stays

住宿表。

字段用途：

- `village_id`：关联 `villages.id`。
- `name`：住宿推荐名称。
- `desc`：描述。
- `price_text`：价格文案。
- `tag`：标签。
- `sort_order`：排序。

## 6. 推荐系统读取方式

未来后端可以这样读取：

1. 读取 `villages + village_profiles`，得到候选村庄及推荐画像。
2. 根据用户输入进行关键词和结构化字段打分。
3. 根据推荐村庄的 `village_id` 读取：
   - `village_routes`
   - `village_foods`
   - `village_stays`
4. 读取 `village_designations` 用于官方背书和权重增强。
5. 组装成当前 `/api/plan` 已兼容的返回结构。

后端返回结构仍可保持：

- `summary`
- `matchScore`
- `recommended`
- `routeOptions`
- `foods`
- `stays`
- `alternatives`
