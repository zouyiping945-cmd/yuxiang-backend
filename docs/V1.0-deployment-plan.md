# V1.0 部署与小程序发布准备方案

## 1. 当前项目状态

- 本地后端已跑通
- Supabase 数据库已接入
- /api/plan 已支持 Supabase 优先读取
- 小程序端本地联调正常
- 当前仍依赖本地 IP，不适合真实发布

## 2. 为什么需要部署后端

微信小程序正式环境不能请求本地 IP，例如：

```text
http://172.20.10.4:3000
```

正式发布需要：

- HTTPS 域名
- 在微信公众平台配置 request 合法域名
- 后端安全保存 `SUPABASE_SERVICE_ROLE_KEY`
- 小程序端只请求公开 HTTPS API，不接触数据库 service role key

因此后端必须部署到线上环境，不能继续依赖局域网地址。

## 3. 推荐部署方案

推荐方案：

- Next.js 后端部署到 Vercel
- Supabase 继续作为数据库
- 小程序端请求 Vercel HTTPS API

优点：

- 适合 Next.js 项目
- 部署快
- 支持环境变量
- 不需要自己维护服务器
- 能直接承载当前 App Router API route
- 后续可以继续保留 Supabase fallback 和本地 seed fallback

## 4. 需要配置的环境变量

后端线上环境需要配置：

```env
SUPABASE_URL=
SUPABASE_SERVICE_ROLE_KEY=
DEEPSEEK_API_KEY=
DEEPSEEK_BASE_URL=
```

说明：

- `SUPABASE_URL`：Supabase 项目基础 URL。
- `SUPABASE_SERVICE_ROLE_KEY`：后端专用 service role key。
- `DEEPSEEK_API_KEY`：如果当前仍需要 DeepSeek provider，则线上也要配置。
- `DEEPSEEK_BASE_URL`：如果当前 provider 依赖自定义 base url，则线上也要配置。

注意：

- `SUPABASE_SERVICE_ROLE_KEY` 只能放后端。
- 不能放到 `miniapp-taro-clean` 前端项目。
- 不能写死到代码里。
- 不能暴露给微信小程序端。

## 5. 小程序端需要改什么

当前 weapp API 地址仍是本地联调地址：

```text
http://172.20.10.4:3000
```

上线后需要改为：

```text
https://your-domain.com
```

建议：

- H5 本地开发继续保留相对路径。
- weapp 生产环境使用线上 HTTPS API。
- 后续可以在 `miniapp-taro-clean/src/utils/env.ts` 中按环境区分本地和生产 API。

## 6. 微信小程序后台需要配置

发布前需要准备：

- 正式 AppID
- request 合法域名
- 隐私协议
- 用户协议
- 小程序名称
- 小程序头像
- 小程序简介
- 类目选择
- 测试账号或体验版说明

注意：

- request 合法域名必须是 HTTPS。
- 域名需要能访问线上后端 API。
- 如果使用 Vercel 默认域名，需要确认微信后台是否允许配置该域名。
- 若后续使用自定义域名，需要完成域名备案、HTTPS 证书和平台配置。

发布前建议逐项检查：

- 小程序端 API_BASE_URL 已切到线上 HTTPS 地址
