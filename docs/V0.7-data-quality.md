# V0.7 数据质量增强说明

## 1. V0.7 目标

V0.7 不继续扩展村庄数量，而是增强现有 24 个郑州圈 seed 村庄的数据质量。

本轮目标是为后续正式数据库迁移做准备，同时让 `/api/plan` 的推荐排序更稳定。前端页面、接口返回结构、Result 页交互和我的行程链路均保持不变。

## 2. 新增字段

`RealVillageData` 新增以下结构化质量字段：

- `recommendedTransport`
- `visitDuration`
- `intensity`
- `elderFriendly`
- `kidFriendly`
- `photoFriendly`
- `foodFriendly`
- `cultureFriendly`
- `wellnessFriendly`
- `selfDriveFriendly`
- `publicTransportFriendly`
- `dataReviewStatus`
- `reviewNotes`

## 3. 字段用途

这些字段用于：

- 更稳定的推荐打分
- 后续 Supabase 表结构设计
- 后续人工核验
- 后续路线、地图、详情页增强
- 为老人友好、亲子友好、拍照友好、农家菜友好等需求建立更明确的结构化判断

当前字段由 seed 数据中的 `tags`、`suitableFor`、`matchKeywords`、`routeOptions` 和 `description` 推导补齐。导出的 24 个村庄都会带上这些质量字段。

## 4. 推荐逻辑增强

V0.7 保持原有关键词打分逻辑，并在其基础上增加结构化字段辅助加分。

当前增强规则：

- 用户提到父母 / 老人 / 不累 / 轻松：
  - `elderFriendly === true` 加分
  - `intensity === "低"` 加分
  - `intensity === "高"` 时，在强调不累场景下扣分
- 用户提到亲子 / 孩子：
  - `kidFriendly === true` 加分
- 用户提到农家菜 / 吃饭 / 美食：
  - `foodFriendly === true` 加分
- 用户提到拍照 / 出片 / 风景：
  - `photoFriendly === true` 加分
- 用户提到传统 / 文化 / 研学：
  - `cultureFriendly === true` 加分
- 用户提到康养 / 放松：
  - `wellnessFriendly === true` 加分
- 用户提到开车 / 自驾 / 周末近郊：
  - `selfDriveFriendly === true` 加分

`matchScore` 仍保持在 88-96 区间，`recommended`、`alternatives`、`routeOptions`、`foods`、`stays` 等返回结构保持兼容。

## 5. 当前限制

- 坐标未补充
- 真实路线未核验
- 真实商家未核验
- 餐饮、住宿、路线仍是保守通用文案
- 当前仍是 seed 阶段
- `dataReviewStatus` 当前统一为 `needs_review`
- 后续需要人工核验后再进入正式数据库

## 6. 下一阶段建议

V0.8 可以做：

- Supabase 表结构设计
- 经纬度补充
- 地图导航能力
- 正式 AppID 与合法域名准备
- 将 `dataReviewStatus` 从 seed 过渡到人工核验流程
- 为每个村庄补充真实路线、停车点、推荐停留时长和适老/亲子标签来源
