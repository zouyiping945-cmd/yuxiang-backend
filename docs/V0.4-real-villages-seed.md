# V0.4 真实村庄 Seed 数据归档

## 1. V0.4 目标

V0.4 的目标是把后端 `/api/plan` 从主要依赖示例 mock 村庄，升级为优先基于河南省乡村旅游地名单中的真实郑州及近郊样本进行推荐。

本轮不接大型数据库，不改前端 UI，不改变 `/api/plan` 返回字段结构。

## 2. 数据来源

数据来源：

- 《附录1 河南省乡村旅游地名单》

本轮仅根据名单中郑州及近郊样本建立第一批 seed 数据。seed 数据用于 MVP 推荐演示和后续数据库迁移前的数据结构验证。

## 3. 为什么只选郑州及近郊

当前小程序 MVP 的核心场景是：

- 周末短途
- 带父母 / 亲子 / 朋友出游
- 轻松慢游
- 农家菜和近郊休闲

郑州及近郊样本更适合第一阶段验证：

- 用户认知成本低
- 距离和路线更容易表达
- 适合 H5 / weapp 演示
- 方便后续人工核验和补充餐饮、住宿、路线数据

## 5. RealVillageData 字段说明

`RealVillageData` 用于描述真实乡村旅游地 seed 数据。

字段包括：

- `id`：系统内唯一标识
- `name`：展示名称
- `province`：省份
- `city`：城市
- `district`：区县
- `town`：乡镇 / 街道 / 管委会
- `village`：村或社区名称
- `fullName`：完整行政区划名称
- `placeLevel`：地点层级
- `rating`：MVP 展示评分
- `distanceText`：保守距离描述
- `tags`：推荐标签
- `suitableFor`：适合人群
- `matchKeywords`：用于规则推荐的关键词
- `description`：推荐描述
- `designations`：官方名单来源记录
- `routeOptions`：查路线 tab 数据
- `foods`：觅美食 tab 数据
- `stays`：寻住处 tab 数据
- `sourceConfidence`：来源可信度
- `dataStatus`：数据状态，当前为 `seed`

## 6. designations 字段说明

`designations` 用于记录样本来自哪个官方名单类别。

字段包括：

- `type`：名单类型，如乡村旅游重点村、美丽休闲乡村、传统村落、康养旅游村
- `sourceName`：来源名称，本轮为《附录1 河南省乡村旅游地名单》
- `sourceIndex`：可选，后续可记录名单序号
- `note`：备注说明

如果同一个村庄后续确认出现在多个名单中，可在 `designations` 中追加多个来源。

## 7. 推荐打分规则

V0.4 在 `build-plan-from-mock.ts` 中优先使用 `REAL_VILLAGES` 进行规则打分。

打分规则：

- 命中 `matchKeywords`：`+12`
- 命中 `tags`：`+10`
- `companions` 命中 `suitableFor`：`+10`
- 老人 / 父母 / 不累 / 轻松：命中老人、轻松慢游或康养休闲：`+15`
- 亲子 / 孩子：命中亲子或亲子游：`+15`
- 农家菜 / 吃饭 / 美食：命中农家菜：`+15`
- 拍照 / 出片 / 风景：命中拍照出片或自然风光：`+15`
- 康养 / 放松：命中康养休闲：`+15`
- 研学 / 文化 / 传统：命中研学、乡村文化或传统村落：`+15`
- 多个 `designations`：额外 `+5`

排序规则：

1. score 从高到低
2. score 相同，`designations` 更多的优先
3. 仍相同，保持 `REAL_VILLAGES` 原始顺序

`matchScore` 会映射到 `88-96` 区间，保持前端显示稳定。

## 8. /api/plan 返回结构

V0.4 保持 `/api/plan` 返回结构不变。

核心字段：

- `summary`
- `matchScore`
- `recommended`
- `routeOptions`
- `foods`
- `stays`
- `alternatives`

字段用途：

- `recommended`：Result 顶部主推荐村庄
- `routeOptions`：查路线
- `foods`：觅美食
- `stays`：寻住处
- `alternatives`：换一个村庄推荐

## 9. 后续扩展计划

后续可分阶段扩展：

1. 扩展到郑州圈 30 个样本
2. 扩展到河南重点村 50 个样本
3. 增加经纬度、路线、停车、开放时间字段
4. 人工核验餐饮、住宿、价格、路线
5. 接入 Supabase 或正式数据库
6. 将 seed 数据迁移为可运营的数据表

## 10. 注意事项

- seed 数据中的餐饮、住宿是保守通用文案。
- 暂不表示真实商家已核验。
- 价格为 MVP 阶段保守表达，例如“人均约 ¥45”“¥198起”“需提前确认”。
- routeOptions 为演示型路线建议，不等同于真实导航时间。
- 不使用远程图片，不返回 Unsplash 图片。
- 后续正式上线前，需要人工核验真实商家、价格、路线和开放状态。
